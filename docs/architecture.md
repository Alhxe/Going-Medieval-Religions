# Architecture

This mod is split into three layers:

```
┌──────────────────────────────────────────────────────────┐
│ 1. Source (git-tracked)                                  │
│    Assets/Mods/ReligionsExpanded/    Unity asset sources │
│    ModBuild/ReligionsExpanded/       JSON + localization │
│    tools/                            build scripts       │
└──────────────────────────────────────────────────────────┘
                            ↓ build
┌──────────────────────────────────────────────────────────┐
│ 2. Compiled artefacts                                    │
│    ModBuild/.../Data/AddressableAssets/*.bundle          │
│    ModBuild/.../Data/Models/*.json (merged from modular) │
└──────────────────────────────────────────────────────────┘
                            ↓ deploy
┌──────────────────────────────────────────────────────────┐
│ 3. Game install                                          │
│    Documents/Foxy Voxel/Going Medieval/Mods/             │
│      ReligionsExpanded/                                  │
└──────────────────────────────────────────────────────────┘
```

## Folder layout

### `Assets/Mods/ReligionsExpanded/` — Unity source

| Folder | Content |
|---|---|
| `Mesh/` | `.fbx` 3D models (cross, pew, altar, bell tower mesh, etc.) |
| `Texture/` | `.png` diffuse/normal/reflection textures |
| `Sprite/` | `.png` UI icons (must have Texture Type = Sprite 2D) |
| `SpriteAssets/` | TMP SpriteAssets generated via Going Medieval menu |

The Unity `AddressableBuilder` window outputs compiled `.bundle` files to `Assets/Mods/ReligionsExpanded/Exported/`. The build script copies those into `ModBuild/.../Data/AddressableAssets/`.

### `ModBuild/ReligionsExpanded/` — final mod

| Path | Purpose |
|---|---|
| `ModInfo.json` | Mod metadata (id, name, version, gameVersion). |
| `Preview.png` | 512×512 image shown in the mods menu. |
| `Data/AddressableAssets/` | Compiled Unity bundles (gitignored — regenerated by build). |
| `Data/Models/<category>/<feature>.json` | Modular game definitions. |
| `Localization/<lang>.json` | Per-language string tables. |

#### Modular JSON categories

| Folder | What goes here |
|---|---|
| `religions/` | Religion configs (christianity, future religions). |
| `buildings/` | Constructable buildings (crosses, pews, altar, confessional, etc.). |
| `items/` | Resources / items (bible, crucifix, robe, wine). |
| `rooms/` | Room types (church, cathedral, bell tower). |
| `research/` | Research nodes. |
| `events/` | Game events / rituals. |
| `stats/` | Custom stats (only if vanilla doesn't cover the need). |

The build script (`tools/build-mod.ps1`) merges each folder into the single repository file the game expects (e.g. `buildings/*.json` → `BaseBuildingRepository.json`).

## Localization strategy

In-game definitions reference **localization keys** (e.g. `"name": "loc_cross_floor_name"`). The actual strings live in `Localization/<lang>.json`. The build script expands keys into the inline `locKeys` format Going Medieval expects, producing one entry per language declared.

This keeps translators editing **one file per language** instead of hunting through every JSON.

See [`translation-guide.md`](translation-guide.md) for details.

## Vanilla interaction

- **Additive**: new IDs only — never overwrite vanilla buildings/rooms/etc.
- **Selective override**: when we need to change a vanilla entry (e.g. unlock `advanced_chapel_christian`), we document why and keep the override minimal.
- **Localization overrides**: e.g. renaming "Restitutionist" → "Christianity" goes through `Localization/<lang>.json` with the same key the vanilla uses.

## Versioning

- Semver: `0.1.0` → `1.0.0` when the mod is feature-complete.
- `ModInfo.json` `modVersion` matches the latest release tag.
- `gameVersion` reflects the targeted Going Medieval version.
